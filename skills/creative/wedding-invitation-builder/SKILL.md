# Wedding Invitation Builder

Build elegant wedding invitation websites with an admin panel. Code the template once, then use the admin panel to input data (names, dates, venues, photos) for each new invitation.

## Architecture

```
wedding-invitation/
├── index.html    → Main invitation page (reads from data.js)
├── admin.html    → Admin panel (form to edit data, outputs data.js)
├── style.css     → Theme styles (gold default)
├── script.js     → Animations, countdown, RSVP, petals
└── data.js       → Wedding data (generated by admin panel)
```

## How It Works

1. **Admin panel** (`admin.html`) has forms for: couple info, events, love story timeline, digital gifts (bank accounts), gallery, music, background (photo/video)
2. Admin fills form → clicks "Generate" → data saved to `window._weddingData`
3. Click "👁 Preview" → opens `index.html` in new tab, data sent via `postMessage` (live preview, no deploy needed)
4. For permanent deploy: copy/download `data.js` → replace in project → deploy to Cloudflare Pages

### Live Preview (postMessage)

Admin and preview communicate via `window.postMessage` to bypass localStorage size limits:

**admin.html** (sender):
```javascript
// Store data on generate
window._weddingData = data;

// Open preview and send data
function openPreview() {
  if (!window._weddingData) { alert('Klik Generate dulu!'); return; }
  previewWin = window.open('index.html', 'previewWin');
  setTimeout(() => {
    previewWin.postMessage({ type: 'weddingData', data: window._weddingData }, '*');
  }, 1500);
}

// Respond to data requests from preview
window.addEventListener('message', (e) => {
  if (e.data.type === 'requestData' && window._weddingData) {
    e.source.postMessage({ type: 'weddingData', data: window._weddingData }, '*');
  }
});
```

**script.js** (receiver):
```javascript
let _liveData = null;
window.addEventListener('message', (e) => {
  if (e.data && e.data.type === 'weddingData' && e.data.data) {
    _liveData = e.data.data;
    loadData(); // re-render everything
  }
});

function loadData() {
  let d = _liveData; // priority: postMessage
  if (!d) { /* try localStorage */ }
  if (!d && typeof weddingData !== 'undefined') d = weddingData; // fallback to data.js
  if (!d) {
    // Request from opener window
    if (window.opener) window.opener.postMessage({ type: 'requestData' }, '*');
    return;
  }
  // Clear dynamic sections before render (prevent duplicates on re-render)
  ['eventsList', 'storyTimeline', 'giftsGrid', 'galleryGrid'].forEach(id => {
    const el = document.getElementById(id);
    if (el) el.innerHTML = '';
  });
  // ... render all sections
}
```

### File Upload Limits

Default upload limits can be raised. Current config: **20MB max** for all uploads (photos, gallery, background photo/video). Files are base64-encoded and embedded in `data.js`.

For the admin panel upload zones, use `FileReader.readAsDataURL()` with size check:
```javascript
if (file.size > 20000000) { alert('File terlalu besar! Max 20MB.'); return; }
```

Update size notes in UI to match (e.g., `<p class="size-note">Max 20MB</p>`).

## Deployment

### Cloudflare Pages (Recommended — Free)

**IMPORTANT: Use manual upload, NOT API token.** API tokens often fail due to permission issues (9106 error).

Steps:
1. Go to `dash.cloudflare.com` → **Workers & Pages** → **Pages** tab
2. Click **"Create a project"** → **"Upload assets"**
3. Project name: `undangan` (or custom)
4. Drag & drop all files from the project folder
5. Click **Deploy**
6. Get link: `https://undangan.pages.dev`

### Custom Domain

After deployment, add custom domain:
1. Pages project → **Custom domains** tab
2. Add domain: `undangan.reviewtechno.me`
3. Cloudflare auto-configures DNS

### Multiple Invitations

For different couples, create separate Pages projects:
- `undangan-ahmad-siti` → one project
- `undangan-budi-ani` → another project
Each has its own `data.js` with different data.

## Pitfalls

- **localStorage has ~5MB limit** — base64-encoded images/videos FAIL SILENTLY when exceeding quota. Do NOT use localStorage for passing large media data between admin and preview. Use `postMessage` instead (see Live Preview section).
- **Cloudflare API Token** deployment often fails with error 9106 — use manual upload instead
- **RSVP data** uses localStorage — lost on browser clear. For persistent RSVP, need backend (D1 database + Worker)
- **Music autoplay** blocked by most browsers — user must click music button
- **Photo URLs** must be publicly accessible (not behind auth)
- **Google Fonts** load from external CDN — can be slow on first visit
- **Custom background NOT showing** — Each section (cover, bismillah, couple, events, story, gifts, gallery, rsvp, footer) has its own opaque `background: var(--bg)` that covers the background layer. Fix: add `body.has-bg` class via JS when background is set, then CSS `body.has-bg .bismillah, .couple, ... { background: transparent !important; }`. Also reduce `.bg-overlay` opacity to ~0.35.
- **localStorage limit for large images** — localStorage has ~5MB limit. Base64 images >3MB fail silently. Use `postMessage` between admin and preview windows instead (no size limit). Admin stores data in `window._weddingData`, sends via `postMessage` to preview tab.
- **Video background upload** — add local file upload option (not just URL) for background video. Store as base64, send via postMessage to preview.
- **Re-rendering** dynamic sections (events, gallery, timeline) — must clear innerHTML before re-populating to prevent duplicates when loadData() is called multiple times
- **Base64 file size** — a 20MB file becomes ~27MB as base64, making data.js very large. For production, prefer URL-hosted media or warn users about load times
- **Custom background NOT showing** — every section (`.bismillah`, `.couple`, `.countdown-section`, `.events-section`, `.story-section`, `.gifts-section`, `.gallery-section`, `.rsvp-section`, `.footer-section`) has opaque `background: var(--bg)` or `var(--bg2)` that COVERS the `.bg-layer` underneath (which has z-index: 0, same as canvas/bokeh layers). Studied indoinvite.com approach: their sections have NO opaque backgrounds — content scrolls over a fixed background image with a semi-transparent overlay on top.

  Fix (3 parts):

  **1. JS — add class when custom background is set:**
  ```javascript
  // In loadData(), inside the background block:
  if (d.background) {
    document.body.classList.add('has-bg');
    // ... rest of background setup
  }
  ```

  **2. CSS — make sections transparent, lighten overlay:**
  ```css
  /* Lighten the overlay (default 0.6 is too dark) */
  .bg-layer.has-image .bg-overlay {
    background: rgba(8, 8, 15, 0.35);
  }
  .bg-layer.has-video .bg-overlay {
    background: rgba(8, 8, 15, 0.3);
  }

  /* Make cover gradient transparent */
  body.has-bg .cover-bg {
    background: transparent;
  }

  /* Make ALL sections transparent so bg shows through */
  body.has-bg .bismillah,
  body.has-bg .couple,
  body.has-bg .countdown-section,
  body.has-bg .events-section,
  body.has-bg .story-section,
  body.has-bg .gifts-section,
  body.has-bg .gallery-section,
  body.has-bg .rsvp-section,
  body.has-bg .footer-section {
    background: transparent !important;
  }

  /* Text shadow for readability over busy backgrounds */
  body.has-bg .section-heading,
  body.has-bg .section-label {
    text-shadow: 0 1px 8px rgba(0,0,0,0.5);
  }
  ```

  **3. Admin panel — background type priority in generate():**
  ```javascript
  background: bgBase64 ? { type: 'image', src: bgBase64 }
    : (bgType === 'video' && bgVideoBase64) ? { type: 'video', src: bgVideoBase64 }
    : (bgType === 'video' && bgVideoUrl) ? { type: 'video', src: bgVideoUrl.trim() }
    : null,
  ```
  Note: bgBase64 only set by file upload. If user uses URL input for background photo, bgBase64 stays empty — need to handle URL case or use bgBase64 for both upload and URL sources.

### Background Video Upload

The background section supports both photo and video uploads from local (max 20MB each). Video section has two options: local upload (base64) and URL input.

**Admin HTML — video upload zone:**
```html
<div id="bgVideoSection" style="display:none;">
  <div class="fg">
    <label>Upload Video Background</label>
    <div class="upload-zone">
      <div class="upload-icon">🎬</div>
      <div class="upload-text">Klik atau drag <span>video di sini</span></div>
      <input type="file" id="bgVideoFile" accept="video/*" onchange="handleBgVideoUpload(this)">
    </div>
    <div id="bgVideoPreviewWrap" style="display:none; margin-top:0.8rem;">
      <video class="gallery-thumb" id="bgVideoPreview" style="width:200px; height:120px; border-radius:8px;" muted></video>
      <button class="rm-gallery" onclick="removeBgVideo()">×</button>
    </div>
    <p class="size-note">Max 20MB. Format: MP4, WebM</p>
  </div>
  <div class="fg">
    <label>Atau Link Video (.mp4)</label>
    <input type="text" id="bgVideoUrl" placeholder="https://example.com/video.mp4" oninput="previewBgVideoUrl(this.value)">
  </div>
</div>
```

**Admin JS — video handler:**
```javascript
let bgVideoBase64 = '';

function handleBgVideoUpload(input) {
  const file = input.files[0];
  if (!file) return;
  if (file.size > 20000000) { alert('Video terlalu besar! Max 20MB.'); input.value = ''; return; }
  const reader = new FileReader();
  reader.onload = (e) => {
    bgVideoBase64 = e.target.result;
    document.getElementById('bgVideoPreview').src = bgVideoBase64;
    document.getElementById('bgVideoPreviewWrap').style.display = 'block';
    document.getElementById('bgVideoUrl').value = '';
  };
  reader.readAsDataURL(file);
}

function removeBgVideo() {
  bgVideoBase64 = '';
  document.getElementById('bgVideoFile').value = '';
  document.getElementById('bgVideoPreviewWrap').style.display = 'none';
}
```
